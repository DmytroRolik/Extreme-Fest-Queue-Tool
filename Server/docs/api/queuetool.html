/service/users post

post

Регистрация нового пользователя


Регистрирует нового пользователя в системе. Поле number является логином пользователя, поле passport его паролем

Пример URL метода:

http://example.com/api/v1/users/


Требования к данным:

  • Поле name должно иметь длинну от 2 до 60 символов, является обязательным
  • Поле surname должно иметь длинну от 2 до 60 символов, является обязательным
  • Поле number должно иметь длинну от 1 до 60 символов, является обязательным
  • Поле passport должно иметь длинну от 2 до 60 символов, является обязательным

Информация об аккаунте


Возвращает полную информацию о текущем аккаунте администратора, включая список доступных ему разрешений на активности и сервис.

  • permission_to_activities - список идентификаторов активностей состояне которых пользователь имеет право изменять (ставить в очередь, удалять из очереди, отмечать опаздавших)
  • permission_to_service - список идентификаторов действий на сервисе, которые пользователь может выполнять (Регистрировать новых пользователей, редактировать расписание...)

    Пример URL метода:
    http://example.com/api/v1/admin/self

/queue/activities/{activityId} get

get

Информация об очереди


Возвращает информацию об очереди на активность. Параметр length указывает какое количество людей находится в очереди на текущий момент. Параметр averageTime показывает среднее время прохождения очереди одним человеком.

В случае если на активности 30 минут нет обновлений, время сбрасывается, и будет возвращено значение -1. Минимальное количество пользователей, после которых будет сформировано среднее время - 3

Пример URL метода:

http://example.com/api/v1/queue/activities/1

get /queue/activities/{activityId}
